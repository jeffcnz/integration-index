Integration Index

Version 0.1

Overview

This has been built to demonstrate a potential proof of concept Integration Index for New Zealand Environmental Data. It has been built using Flask linked to a Postgesql database using SQLAlchemy. Flask provides a simple API and SQLAlchemy means that if a different database engine is required the migration will be easy.

The goal was to develop an api that is able to provide a single, consistent OGC compliant service that uses national site identifers and parameter names to retrieve time series data from a variety of agencies that use different time series servers and have varying naming conventions.

The api allows:

  • A single entry point to environmental data with consistent naming and functionality.
  • Features to be tested that are not available on all commercial systems. (e.g. JSON output)
  • Services to be developed to use the data, based on consistent naming.

SOS capabilities.

  • GetCapabilities
  • GetFeatureOfInterest
  • GetObservation

Viewer

A more userfriendly method of exploring the data availability is provided via these web-pages. Use the sites page to select a site of interest and then select the data on the site page.
